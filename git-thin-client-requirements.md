# Git Thin Client — 要件定義書

## 1. プロダクト概要

**コンセプト:** Sourcetreeの重さ・複雑さを捨て、自分のワークフローだけに最適化した軽量Gitダッシュボード

| 項目 | 内容 |
|---|---|
| 技術スタック | Next.js ローカルWebアプリ |
| 対象ユーザー | 自分専用（シングルユーザー） |
| 置き換え対象 | Sourcetree |
| 連携先 | GitHub（PR・レビューはWeb UIに委譲）、VSCode（コンフリクト解消を委譲） |
| テーマ | OS/ブラウザ設定に追従（light/dark） |
| 操作方針 | キーボードショートカット必須＋マウス併用 |

---

## 2. 対象リポジトリ

- モノレポ × 約5個を日常的に操作
- **タブUI**でリポジトリを切り替える（ブラウザタブではなくアプリ内タブ）
- タブごとに独立した状態（ブランチ、diff、ステージング等）を保持

---

## 3. ブランチ戦略前提

- **GitHub Flow**（mainから切ってPR）
- developブランチ等は使わない
- PRの作成・レビューはGitHub Web UIで行う（本ツールのスコープ外）

---

## 4. コアワークフロー

日常の作業フローを最短経路で実現する。

```
[pull main] → [merge main into current branch] → [コード変更] → [diff確認] → [hunk単位staging] → [commit] → [push]
```

### 4.1 Pull & Merge（最頻出操作）

- **ワンアクション**で `pull origin main` → `merge main into current branch` を実行
- コンフリクト発生時はその旨を表示し、VSCodeで開くボタンを提供（解消自体は委譲）
- コンフリクト解消後の状態を検知して続行を促す

### 4.2 ステージング & Diff確認

**レイアウト: 3ペイン構成**

```
┌─────────────────┬──────────────────────────────┐
│  変更ファイル    │  選択ファイルのhunk diff      │
│  リスト          │                              │
│                  │  □ hunk 1                    │
│  ■ file-a.ts    │  ☑ hunk 2  [lines 14-29]     │
│  ☐ file-b.ts    │  □ hunk 3  [lines 33-40]     │
│  ☑ file-c.ts    │  (Shift+click for range)     │
└─────────────────┴──────────────────────────────┘
```

**左ペイン: 変更ファイルリスト**

- チェックボックスは **3ステート**:
  - `☐` 未stage（全hunk未選択）
  - `☑` 全stage（全hunk選択済み）
  - `■` **部分stage（一部hunkのみ選択 → 黒塗りつぶしindeterminate）**
- ファイルのチェックをクリック → 全hunkをまとめてstage/unstage
- indeterminate状態からクリック → 全選択にする（再クリックで全解除）

**右ペイン: Hunk Diff表示**

- 選択したファイルのdiffをhunk単位で表示
- 各hunkにチェックボックス（stage/unstage）
- **複数行まとめてチェック（必須）:**
  - Shift+クリックで範囲選択（hunk内の連続する行をまとめて選択）
  - ドラッグ選択 → 選択した行だけをstage対象にする
  - これにより1つのhunkをさらに細かく分割してstageできる（`git add -p` の `s`(split) + `e`(edit) 相当）
- キーボードで上下移動、スペースでstage/unstageトグル

### 4.3 コミット

- 現在のブランチ名を常時表示
- コミットメッセージ入力欄
- 2つのコミットアクション:
  - **「Commit」** — 現在のブランチにコミット
  - **「Commit to New Branch」** — ブランチ名入力 → ブランチ作成＋チェックアウト＋コミットをワンアクションで実行
- Commit後の自動pushはオプション（トグル設定）

### 4.4 Push

- unpushedコミットがある場合、目立つインジケーターで表示
- ワンクリック / ショートカットでpush

---

## 5. ブランチ管理

### 5.1 作成・切り替え

- コミットフローに統合（4.3参照）
- ブランチ切り替えはコマンドパレット風UI or リストから選択
- 切り替え時にuncommitted changesがある場合は警告

### 5.2 マージ済みブランチの掃除

- マージ済み（リモートで削除済み）ブランチを**自動検知**
- 一覧表示 → 一括削除ボタン
- 定期チェック or タブアクティブ時に検知

---

## 6. コミット履歴 & 検索

### 6.1 ブランチグラフ

- Sourcetree風のブランチツリー表示
- 各コミットの1行サマリー、author、日時
- コミットをクリック → 変更ファイル一覧 & diffを表示

### 6.2 検索機能（3種類）

| 検索タイプ | 入力 | 結果 |
|---|---|---|
| コミットメッセージ検索 | キーワード | 該当コミット一覧 |
| ファイル履歴検索 | ファイルパス（部分一致/glob） | そのファイルの変更コミット一覧 |
| コミットID検索 | SHA（前方一致） | 該当コミットにジャンプ |

- 統合検索バー（コマンドパレット風）から全種類にアクセス
- キーボードショートカットで即座に検索バーにフォーカス

### 6.3 コミット操作

- 特定コミットへのcheckout（detached HEAD警告付き）
- 特定コミットのrevert（revertコミット生成）

---

## 7. ステータス & ポーリング

### 7.1 表示情報

各リポジトリタブに常時表示:

- 現在のブランチ名
- unpushed / unpulled コミット数
- mainとの差分コミット数（ahead/behind）
- 変更ファイル数

### 7.2 更新戦略

- **タブアクティブ時:** 定期ポーリング（間隔は設定可能、デフォルト30秒程度）
- **タブ非アクティブ時:** ポーリング停止
- `document.visibilitychange` イベントで制御

---

## 8. キーボードショートカット

最低限定義するショートカット:

| 操作 | ショートカット例 |
|---|---|
| 検索バーを開く | `Cmd+K` or `Cmd+P` |
| コミット実行 | `Cmd+Enter` |
| Push | `Cmd+Shift+P` |
| Pull & Merge main | `Cmd+Shift+M` |
| ファイルリスト↑↓移動 | `j` / `k` |
| Stage/Unstageトグル | `Space` |
| リポジトリタブ切り替え | `Cmd+1~5` |
| ブランチ切り替え | `Cmd+B` |

- ショートカット一覧をヘルプモーダルで表示（`?`キー）
- 将来的にカスタマイズ可能にする余地を残す

---

## 9. スコープ外（明示的に除外）

- PR作成・レビュー → GitHub Web UI
- マージコンフリクトの解消 → VSCode
- リベース操作
- サブモジュール管理
- Git LFS操作
- マルチユーザー対応
- リモートリポジトリの管理（clone等）

---

## 10. 非機能要件

| 項目 | 要件 |
|---|---|
| 起動速度 | Sourcetreeより明確に速いこと |
| 応答性 | diff表示・ブランチ切り替えは体感即座 |
| モノレポ対応 | 巨大リポでもUIがブロックしない（git操作は非同期） |
| テーマ | `prefers-color-scheme` 追従 |
| ローカル動作 | インターネット不要（git操作のみネットワーク使用） |

---

## 11. 技術アーキテクチャ（方針メモ）

```
┌──────────────────────────────────────┐
│  Next.js Frontend (localhost:3000)   │
│  - React UI                          │
│  - キーボードショートカット管理       │
│  - ポーリング制御                     │
├──────────────────────────────────────┤
│  Next.js API Routes (Backend)        │
│  - git コマンド実行（child_process）  │
│  - リポジトリパス設定管理             │
│  - diff / log パース                 │
├──────────────────────────────────────┤
│  Local File System                   │
│  - ~/.git-thin-client/config.json    │
│  - リポジトリパス一覧                 │
└──────────────────────────────────────┘
```
